### 상세 속성 (Attributes)
#### 개요
목적
- ERP 기반 원가 시스템의 정합성 확보, 운영 부문 실적의 원천 거래 추적(Tracking)을 위해 
- ERP 내 운영 거래(생산, 사육 등)으로 우선 실적 처리 후 원가결산 수행


내용
- 원가계산 방법
  1) 제조원가 : SAP CO 표준기능 준용 (자재원장(Material Ledger) 기반 실제원가 계산(Actual Costing))
  2) 사육원가 : 운영 부문 거래 실적(물량)은 SAP CO 자재원장(Material Ledger)을 활용하되, 
                   실제원가 계산은 사육원가 특성과 논리를 기준으로 별도 개발 (원가결산 마감 회계처리 포함)
- 사육 및 생산 부문의 운영 시스템(MES or Legacy)에서 처리한 거래 실적은 반드시 각 운영 업무 영역(PP, MM, SD, LE)에서 
  1차적으로 I/F(인터페이스) 하여 정상 실적(Production Order 단위 생산입고, 생산투입 및 작업공수, 생물자산 입고 및 출고) 처리 진행
- 제조원가와 사육원가 모두 ERP 운영 실적을 기반으로 하는 표준 자재원장(Material Ledger) 기능을 우선 사용하고자 함.
  (운영 원천 거래 추적(Tracking) 용이)
- 미래 유지보수 측면에서도 SAP의 지속적인 변화 사항을 최대한 수용할 수 있도록 CO 자재원장 표준 구조를 유지하는 것이 바람직함.
  (표준 기능을 준용하는 제조원가와 개발에 의한 사육원가의 표준 재고수불(자재원장) 구조는 동일하게 구성)


시스템 구현방안
- SAP 표준 물동 흐름(생산 실적, 재고 입출고 실적 등)을 준용하여 CO 자재원장(Material Ledger) 구성
- 사육원가는 자재원장의 물량 정보를 근간으로 제조원가 계산 및 재고자산 수불 편성이 이루어지도록 개발


연계 시스템
- ERP 內 MM(구매자재), PP(생산), SD(영업), LE(출하)
- 사육원가 관련 운영시스템 : 가금(HATIS), 양돈(PigOn), 한우(한우방)
