### 상세 속성 (Attributes)
#### 개요
I/F 되어 온 내역을 조회를 하고 추가로 필요한 데이터를 입력 후 BP 를 생성 한다. 이때 조견표를 참조 하여 등록 되므로 반드시 세부 내역을 확인 해야 한다.
사업자등록번호가 유효 한 것인지 등을 확인 하는 것은 e-ACCOUNTING(HOMETEX) 와 확인
그룹웨어에서 결재가 난 내역이 SAP CBO Table 에 I/F 되어 오면, 이를 확인 한다. 문제가 있으면 다시 품의를 하거나, I/F 를 다시 해야 한다 등록된 내역이 타사가 공유 되어서는 안된다. 최초 등록 한 회사만 볼 수 있도록 한다.

- SAP BP 는 많은 필드 들이 산재해 있어, BP 기본 화면으로 등록 하는 것은 어려움
 (Main BP 를 BP 에 직접 등록 하지 않고, CBO 프로그램과 조견표를 이용하여 쉽게 등록 한다)
 (Groupware 에서 I/F 되어 온 것은 내역을 기본값으로 보여 주고 등록을 한다)
 (국가별 Unique Key 를 확인 한다)
 (계좌는 Customer 는 회계에서 등록관리, 구매는 구매팀에서 등록 관리)
 (법인공통카드는 ID-TYPE 등으로 구분 가능 여부 확인)
 (계좌가 변경 되는 경우는 자금팀 합의 Flow 추가)
